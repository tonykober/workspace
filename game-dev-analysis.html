核心判斷

團隊技術能力完全足夠。30 隻現有角色完全覆蓋新遊戲 8 種體型需求，不需要新建角色模型或骨架。挑戰在於「風格轉換」— 透過改貼圖將恐怖風格轉為歡樂風格。

製作方向三要點

  1. 圓潤化 — 所有造型去掉尖角、複雜配件，強調球形/膠囊形基底
  2. 提亮化 — 背景冷色低飽和，角色暖色高飽和，特效最高飽和
  3. 減法設計 — 55 吋螢幕 1~2 公尺觀看距離，細節看不到，輪廓和色塊才是重點

風格參考標竿

  • Fall Guys 的圓潤感
  • Brawl Stars 的色彩飽和度
  • Splatoon 的街頭潮流感
  • 目標：讓 6 歲覺得好玩，12 歲覺得不丟臉

需要的心態轉換

團隊習慣做「細節豐富」的角色，新專案要學會「少即是多」：

  • 減少配件裝飾 → 強化輪廓辨識
  • 減少材質細節 → 強化色塊對比
  • 減少暗色調 → 強化明亮歡樂感

色彩規範

  • 背景層：冷色調 / 飽和度 30~50% / 明度 40~60%
  • 角色層：暖色調 / 飽和度 70~90% / 明度 60~80%
  • 特效層：任意色相 / 飽和度 90~100% / 明度 80~100%

角色視覺規範

  • 剪影測試：每個角色純黑剪影必須一眼可辨
  • 主色分配：8 角色各有專屬主色（紅/藍/綠/黃/橙/紫/粉/白）
  • 體型差異：矮胖/高瘦/方正/不定形/長腿/小重/大輕/帶刺
  • 表情系統：大眼睛 + 3~4 種表情切換（開心/驚嚇/生氣/暈眩）
  • Squash & Stretch：所有動作加入擠壓拉伸變形
  • 面數預算：3,000~5,000 面/角色
  • 貼圖：1024×1024，手繪色塊+漸層

場景視覺規範

  • 可讀性優先：地面/牆壁/障礙物色彩分明，不混淆
  • 機關高亮：可互動物件用高飽和色+微發光標示
  • 安全區標示：場地邊緣用明確視覺提示（虛線/漸變）
  • 動態元素：傳送帶、旋轉台用動畫+粒子強調運動方向
  • 面數預算：單地圖 50,000~80,000 面

特效視覺規範

  • 打擊回饋：速度線 + 星星 + 震動 + 短暫慢動作
  • 得分回饋：數字彈出 + 光圈擴散 + 音效同步
  • 連擊系統：combo 數字越大，特效越誇張（顏色升級、粒子增多）
  • 淘汰特效：被推出場外的誇張飛行軌跡 + 爆炸星星

UI 視覺規範

  • 圓角矩形為基礎形狀
  • 高對比色（深底+亮字 或 亮底+深字）
  • 字體大、粗，1~2 公尺外可讀
  • 動態過場（滑入/彈出/縮放）

角色設計規範

身體比例

  • 頭身比：2~2.5 頭身
  • 頭部佔比：約佔身高 40-45%
  • 四肢：短粗圓潤，無明顯關節
  • 手部：圓球手或三指手套手
  • 腳部：短粗，圓柱或簡化鞋型

輪廓辨識度

  • 純黑剪影狀態下可被辨認
  • 頭部造型為主要辨識特徵
  • 至少 3 種體型分佈（圓胖/標準/瘦高）

表情系統（每角色必備）

  • 預設（開心微笑）
  • 衝刺（興奮/專注）
  • 受擊（誇張驚訝）
  • 暈眩（螺旋眼）
  • 勝利（大笑/歡呼）
  • 落敗（沮喪但可愛，不能哭泣）

每隻角色需提供 Turnaround：正面/45度/側面/背面 + T-Pose + 表情表 + 配色變體

模組化規則

  • 模組尺寸基準：1 格 = 角色身高
  • 平台最小寬度：2 格
  • 所有可互動物件需有視覺提示（發光邊緣/微動畫/色彩突出）

材質風格

乾淨卡通（Clean Toon），大面積平塗+柔和漸層，禁止寫實材質

光照

  • 45 度頂側光柔和無硬陰影
  • 暗部不低於 40% 亮度
  • 角色額外 Rim Light
  • 禁止全黑區域

動畫風格指引

  • Squash & Stretch：高度使用（變形量 20-30%）
  • 預備動作：明顯，讓玩家可預判
  • 跟隨動作：耳朵/尾巴/配件延遲跟隨
  • 節奏：快進慢出，打擊瞬間 1-2 幀定格

打擊感

  • 擊中白色閃幀 + 畫面微震
  • 擊飛弧線拖尾 + 速度線
