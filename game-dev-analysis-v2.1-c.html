C1. 美術風格指南

風格化程度定義

維度程度說明
比例誇張度高（2~2.5 頭身）大頭短身，四肢圓潤粗短
邊緣處理圓滑無銳角所有邊緣倒圓角，無尖刺造型
材質簡化中高色塊+漸層為主，少量手繪紋理點綴
細節密度低大面積留白，重點在臉部與配件
光影風格卡通渲染明確明暗分界，柔和環境光

DO / DON'T

DO ✅
  • 圓潤、膨脹、有彈性感的造型
  • 明亮飽和的色彩
  • 簡潔清晰的輪廓剪影
  • 誇張但友善的表情

  • 過度複雜的細節堆疊
  • 恐怖、驚嚇的表情

UI 色盤

  • 主要操作：#4D96FF（藍）
  • 確認/正面：#6BCB77（綠）
  • 警告/計時：#FFD93D（黃）
  • 取消/危險：#FF6B6B（紅）
  • 背景：#FFFFFF / #F8F9FA
  • 文字：#2D3436（深灰，非純黑）

色彩使用規則

  • 60-30-10 法則：主色 60%、輔色 30%、點綴色 10%
  • 對比度要求：文字與背景對比度 ≥ 4.5:1（WCAG AA）
  • 角色辨識：每隻角色在縮小至 64px 時仍可透過色彩區分
  • 兒童安全：避免大面積高頻閃爍、避免紅綠直接相鄰（色盲友善）

色彩情緒對應

情境色彩方向範例
歡樂/勝利暖色高飽和 + 金色粒子黃橘紅漸層爆發
危險/倒數紅橘脈衝 + 邊緣暗角畫面邊緣紅色閃爍
加速/強化青藍流線 + 白色拖尾角色周圍藍色光環
暈眩/受擊黃色星星 + 螺旋線卡通式暈眩符號

角色設計規範

身體比例

  • 頭身比：2~2.5 頭身
  • 頭部佔比：約佔身高 40-45%
  • 四肢：短粗圓潤，無明顯關節
  • 手部：簡化為圓球手或三指手套手
  • 腳部：短粗，可為圓柱或簡化鞋型

輪廓辨識度

  • 每隻角色的剪影必須在純黑填充狀態下可被辨認
  • 頭部造型（耳朵/角/帽子/髮型）為主要辨識特徵
  • 體型需有差異化：圓胖型 / 標準型 / 瘦高型（至少 3 種體型分佈）
  • 主色分配：8 角色各有專屬主色（紅/藍/綠/黃/橙/紫/粉/白）

表情系統（每角色必備）

  • 預設（開心微笑）
  • 衝刺（興奮/專注）
  • 受擊（誇張驚訝）
  • 暈眩（螺旋眼）
  • 勝利（大笑/歡呼）
  • 落敗（沮喪但可愛，不能哭泣）

規則：以眼睛+嘴巴變化為主，眉毛為輔，不使用寫實肌肉變形。大眼睛 + Squash & Stretch 所有動作加入擠壓拉伸變形。

場景設計規範

6 張地圖設定

地圖主題色調方向核心機關
1糖果工廠粉紅/奶油/巧克力輸送帶、彈跳果凍
2海洋派對藍/青/珊瑚浮板、水流推力
3太空站深紫/霓虹藍/銀低重力區、彈射管
4叢林遺跡綠/金/土色藤蔓擺盪、崩塌平台
5玩具城堡紅/藍/黃（三原色）積木牆、彈珠軌道
6雲端樂園白/淡藍/彩虹消失雲朵、風力推送

場景視覺規則

  • 可讀性優先：地面/牆壁/障礙物色彩分明，不混淆
  • 機關高亮：可互動物件用高飽和色+微發光標示
  • 安全區標示：場地邊緣用明確視覺提示（虛線/漸變）
  • 動態元素：傳送帶、旋轉台用動畫+粒子強調運動方向

模組化規則

  • 每張地圖由可重組模組構成（平台、牆壁、機關、裝飾）
  • 模組尺寸基準：1 格 = 角色身高 ≈ 遊戲內 2m
  • 平台最小寬度：2 格（可站立 2 隻角色）
  • 攝影機可視範圍：約 16×9 格
  • 地圖總面積：約 20×12 格（含畫面外緩衝區）
  • 所有可互動物件需有視覺提示（發光邊緣/微動畫/色彩突出）
🖼️ 場景比例尺參考圖

材質風格

  • 主要風格：乾淨卡通（Clean Toon）
  • 大面積平塗色塊 + 柔和漸層
  • 少量手繪紋理用於地面/牆面增加質感
  • 禁止寫實材質（木紋、金屬反射等需風格化處理）
  • 所有材質需在無光照狀態下仍可辨識物件

光照風格

  • 主光源：45 度頂側光，柔和無硬陰影
  • 環境光：充足，暗部不低於 40% 亮度
  • 角色光：額外 Rim Light 確保角色從背景跳出
  • 禁止：全黑區域、恐怖氛圍光照、過度體積光

動畫風格指引

動作原則

  • Squash & Stretch：高度使用（變形量 20-30%）
  • 預備動作（Anticipation）：明顯，讓玩家可預判
  • 跟隨動作（Follow Through）：耳朵/尾巴/配件需有延遲跟隨
  • 動作節奏：快進慢出（Ease In/Out），打擊瞬間可用 1-2 幀定格

打擊感表現

  • 擊中瞬間：畫面微震 + 白色閃幀（1 幀）+ 短暫慢動作
  • 擊飛軌跡：弧線拖尾 + 速度線
  • 落地：Squash 壓扁 + 灰塵粒子
  • 得分回饋：數字彈出 + 光圈擴散 + 音效同步
  • 連擊系統：combo 數字越大，特效越誇張（顏色升級、粒子增多）
  • 淘汰特效：被推出場外的誇張飛行軌跡 + 爆炸星星
  • 禁止：流血、骨折、寫實痛苦表情

物理互動動畫原則

  • 被推動時身體需有傾斜反應
  • 碰撞後彈開需有旋轉
  • 抓取物件時手部需有吸附動畫（非瞬間切換）

特效視覺規範

  • 打擊回饋：速度線 + 星星 + 震動 + 短暫慢動作
  • 得分回饋：數字彈出 + 光圈擴散 + 音效同步
  • 連擊系統：combo 數字越大，特效越誇張
  • 淘汰特效：被推出場外的誇張飛行軌跡 + 爆炸星星

UI/UX 視覺規範

  • 字體：圓體粗體（標題）、圓體中等（內文）、最小 36pt
  • 圖標：圓角矩形或圓形底框、線條 4px @1080p、填充式優先
  • 按鈕：圓角矩形（radius 50% 高度）、有明確按下狀態（下沉+變暗）
  • HUD：玩家資訊固定四角、中央淨空、半透明底板確保可讀性
  • 動態過場：滑入/彈出/縮放
  • 高對比色：深底+亮字 或 亮底+深字
  • 字體大粗：1~2 公尺外可讀
  • 倒數計時：最後 10 秒放大脈衝動畫
🖼️ HUD 佈局示意圖

常見問題 FAQ

  • Q: 角色可以有武器嗎？ → 可以，但需為玩具化/卡通化造型（如充氣錘、水槍）
  • Q: 場景可以有文字嗎？ → 盡量用圖像符號取代文字，若必要則用簡單英文/注音
  • Q: 血量怎麼表現？ → 不使用血條，改用「生命數」或「護盾光環」視覺

命名細則

  • 全英文，PascalCase（每個詞首字母大寫）
  • 編號用兩位數（01, 02...）
  • 變體用顏色或描述詞（Red, Blue, Broken, Open）
  • 工作中加 _WIP，審核中加 _Review
  • 禁止空格、中文、特殊符號

交付流程

  1. 完成作業 → 自我檢查（對照下方 Checklist）
  2. 上傳至指定共享資料夾（依命名規則放入對應目錄）
  3. 填寫交付清單表格（標註完成項目）
  4. 通知美術主管進行審核
  5. 審核通過 → 標記為 Final / 審核不通過 → 修改後重新提交（最多 3 輪）

版本管理

  • _WIP：工作中
  • _Review：審核中
  • 無後綴：正式版
  • _Archive 資料夾：歷史版本
