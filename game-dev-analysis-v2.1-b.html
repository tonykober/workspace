區塊 B：新遊戲製作規劃

B1. 遊戲概述與視覺目標

專案概述

  • 類型：1~4 人物理亂鬥街機遊戲
  • 平台：大型電玩街機
  • 目標受眾：6~12 歲兒童 + 陪同家長（25~45 歲）
  • 場景：家庭娛樂中心、遊樂場、親子餐廳
  • 開發期：6 個月
  • 規模：6 可玩角色 / 5 張地圖

核心體驗關鍵字

歡樂 混亂 可愛 彈性 派對感 安全 親子

玩法概述

一句話描述：1~4 人 3D 物理亂鬥街機遊戲，以物理引擎為核心，每回合隨機切換不同玩法模式，製造不可預測的歡樂混亂。

核心循環：

投幣 → 選角色(8秒) → 回合1(隨機模式) → 回合2(隨機模式) → 回合3(隨機模式) → 總分結算

遊戲模式（5 種）

模式類型規則時間
互推淘汰物理亂鬥推出場外即淘汰60~90 秒
障礙賽跑闖關競速跑到終點，最後者淘汰45~60 秒
搶旗混戰搶奪持有持旗累積秒數60 秒

  • 操作：搖桿移動 + A 攻擊 + B 跳躍，30 秒內學會。碰觸自動互動（撿道具/旗幟/金幣）。
  • 計分：每回合：1名=3分、2名=2分、3名=1分、4名=0分。3 回合總分決定最終排名。
  • 單局時間：約 2.5~3 分鐘（含結算），高翻桌率。

視覺目標

目標：讓 6 歲覺得好玩，12 歲覺得不丟臉，家長願意一起玩

製作方向三要點

  1. 圓潤化 — 所有造型去掉尖角、複雜配件，強調球形/膠囊形基底
  2. 提亮化 — 背景冷色低飽和，角色暖色高飽和，特效最高飽和
  3. 減法設計 — 55 吋螢幕 1~2 公尺觀看距離，細節看不到，輪廓和色塊才是重點

硬體效能限制

本遊戲運行於 Android 平台，美術製作須遵守硬體效能規範：

  • 場景總三角面 ≤ 80K（含角色）
  • Draw Calls ≤ 100
  • 粒子總數 ≤ 400
  • 燈光 1 盞
  • 目標 40 FPS

B2. 風格轉換分析

核心判斷

團隊技術能力完全足夠。30 隻現有角色完全覆蓋新遊戲 6 種體型需求，不需要新建角色模型或骨架。挑戰在於「風格轉換」— 透過最低程度調色去除血腥恐怖元素。

新舊風格差異對照

項目桑比槍台（現有）新街機遊戲（目標）
色調暗色系（紫/深綠/黑）明亮高飽和（橙/黃/藍/綠）
角色比例Q 版但仍有細節（1:1.5~1:2）更圓潤極簡（1:1~1:1.5）
貼圖風格手繪寫實質感手繪色塊+漸層，減少細節
輪廓複雜剪影（帽子/披風/武器）簡潔圓潤剪影（遠距離辨識）
面數5,000~10,000 面2,000~3,500 面
情緒陰森/詭異/華麗歡樂/搞笑/酷炫
目標受眾青少年~成人6~12 歲兒童 + 陪同家長

需要的心態轉換

團隊習慣做「細節豐富」的角色，新專案要學會「少即是多」：

  • 減少配件裝飾 → 強化輪廓辨識
  • 減少材質細節 → 強化色塊對比
  • 減少暗色調 → 強化明亮歡樂感

NPC 體系

場景 NPC（敵對/中立）— 3 隻

NPC定位出現模式行為視覺特徵動畫需求
搗蛋鬼場景干擾者收集狂熱、障礙賽跑撒金幣/擋路/被撞跌倒比玩家小一號，螢光色+搗蛋面具3 組

裁判規則提示+氣氛所有模式通用開場吹哨/淘汰誇張反應黑白條紋衣+哨子3 組

跟班 NPC（協力）— 4 隻

  • 觸發：玩家撿到召喚道具（召喚鈴鐺/友誼球）
  • 持續：10~15 秒
  • 行為：自動追蹤敵方推擊 / 幫擋一次攻擊 / 幫搶旗幣後交給玩家
  • 退場：時間到自動「彈飛離場」（搞笑退場）
  • 視覺：頭上顯示愛心/星星標記
  • 動畫：每隻 2 組（跟隨跑步+攻擊）

演出裝飾角色 — 3 隻

  • 用途：場景背景觀眾、待機畫面演出、回合過場裝飾
  • 視覺：改貼圖為觀眾風格（加圍巾/旗幟/爆米花）
  • 動畫：每隻 1~2 組（歡呼循環、舉牌）

效能限制

  • 同時在場 NPC 不超過 2~3 隻
  • 跟班召喚：同時最多 1~2 隻跟班在場
  • 特效預算：NPC 相關特效控制在場景機關預算（≤ 120 粒子）內
  • NPC/跟班/演出角色使用 VAT（Vertex Animation Texture）+ GPU Instancing 技術，確保效能流暢

隱性平衡機制

  • 連勝者下回合移動速度微降 5~10%
  • 體型大角色 = 目標大 = 更容易被打中
  • 讓小孩有贏的機會，家長不覺得被放水

投幣機制調整

  • 親子套票：2 幣 = 2 人同時開始

  • 台面高度 80~85cm + 兒童腳踏墊標配
  • 搖桿面板微傾斜 10~15 度

  • 週末家庭客成為主力消費群
  • 家長從被動付費→主動參與
  • 續幣率提升（親子互動樂趣驅動）
