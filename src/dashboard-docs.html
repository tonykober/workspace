資料更新機制

每日排程

時間任務說明
08:00advisor 提供台股資料JSON 格式，含所有追蹤個股
08:30台股操盤日報建立當日 data json（僅 stock_briefing），git push
09:00全分類資訊更新搜尋 9 分類最新資訊，補入當日 json，git push

更新流程（08:30 台股）

  1. 讀取 Sheet watchlist 確認完整清單
  2. 建立 data/YYYY-MM-DD.json（僅含 stock_briefing，其他分類為空）
  3. 更新 data/index.json
  4. git push
  5. 在 topic 2497 發概要

更新流程（09:00 全分類）

  1. 搜尋 AI、遊戲、科技、加密、開發、美術、設計、商業、財經
  2. 填入當日 json 的各分類 + summaries
  3. git push
  4. 在 topic 2497 發概要

週末/國定假日

  • 台股操盤：跳過
  • 全分類資訊：照常更新

  • 追蹤清單（含 NAV/PB/折溢價/PE/離季線/建議標籤）

  • searched 陣列：已搜尋但未找到 → 顯示「查無此個股」
  • 趨勢顯示：▲紅（漲）、▼綠（跌）、▬灰（平）+ 描述文字

規則與限制

Watchlist 同步

  • Sheet 為唯一真實來源（Single Source of Truth）
  • 前端載入時讀取 Sheet 覆蓋 localStorage
  • 保護邏輯：如果 Sheet 含 "test" 開頭項目，不覆蓋本地
  • 新增/刪除同時更新 localStorage 和 Sheet

台股操盤

  • 週末/國定假日跳過
  • 每日 08:30 發布，僅含 stock_briefing（其他分類留空）
  • 09:00 補入全分類資訊
  • 所有 watchlist 個股必須有資料（無則標記在 searched 陣列）
  • updated_at 欄位記錄最後更新時間

資訊內容

  • 使用中文
  • 連結必須指向具體文章內頁（非首頁）
  • 不發到 AI 專家（topic 64）或遊戲專家（topic 65）頻道
  • 發布到儀表板頻道（topic 2497）

股票代碼規則

  • 保留前導零（0056 不是 56）
  • 可含字母（00982A、00981A、00403A）
  • 匹配邏輯：精確代碼前綴優先，模糊 includes 需長度≥4

趨勢顯示

  • ▲ 紅色（漲）
  • ▼ 綠色（跌）
  • ▬ 灰色（平）
  • trendArrow(t) 函數：去除前導漲/跌/平字元，保留描述
